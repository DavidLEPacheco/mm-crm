# Mazar Martin CRM — Pipeline

Python pipeline that scrapes property data, parses real estate emails, and
injects everything into the master HTML (`mazar_martin_app.html`). The output
of the pipeline becomes the deployed `index.html` at the repo root, served
via GitHub Pages.

This folder is the future Railway service — it's organised so it can be
deployed standalone once we move scheduling off the local machine.

## Folder layout

```
pipeline/
├── scripts/                Python scripts (tracked in git)
├── data/                   Runtime JSONs (gitignored — at pipeline/ root currently)
├── master/                 mazar_martin_app.html lives here (gitignored)
├── requirements.txt
├── .env.example            Template — copy to .env (gitignored) for credentials
└── README.md
```

Note: per Gerard's original setup, the scripts expect data JSON files to live
one directory above `scripts/` — i.e. directly in `pipeline/`. The same is
true for `mazar_martin_app.html`. So in practice you'll see runtime files at
`pipeline/*.json` and `pipeline/mazar_martin_app.html` — these are gitignored.

## Local setup (Windows / cross-platform)

```powershell
# From the pipeline/ folder
python -m venv .venv
.\.venv\Scripts\Activate.ps1     # Windows PowerShell
# source .venv/bin/activate       # macOS/Linux

pip install -r requirements.txt
playwright install chromium       # ~150 MB download
```

## Credentials

The Gmail scraper authenticates via Gmail App Password (IMAP), not OAuth.

Set up either way:

**Option A — env var:**
```powershell
$env:GMAIL_APP_PASSWORD = "xxxx xxxx xxxx xxxx"
```

**Option B — `.mm_credentials` file** (in `pipeline/scripts/`, gitignored):
```
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
```

The App Password is generated by Gerard at
`myaccount.google.com → Security → App Passwords` for `mazarmartinapp@gmail.com`.

## Running the pipeline

The original Mac orchestrator chained these scripts (see
`pythonfiles/lns_agents_scripts/_run_scrape_wash_deploy.command` for the
canonical sequence):

1. `scrape_gmail.py` — pull Proping + off-market emails
2. `scrape_agency_websites.py` — Atlas, McGrath, Stone, etc.
3. `scrape_onthehouse.py` — OnTheHouse listings
4. `scrape_domain_realestate.py` — Domain.com.au listings + sold
5. `inject_email_data.py` — bake email data into HTML
6. `wash_properties.py` — fill missing beds/baths/types from caches
7. `fill_forsale.py` / `fill_sold_prices.py` / `fill_sold_fields.py` /
   `fill_last_sold.py` / `fill_proping_fields.py` — field fillers
8. `inject_agency_offmarket.py`
9. `dedup_offmarket.py`

Output: updated `pipeline/mazar_martin_app.html`. Copy to repo root as
`index.html` to deploy.
